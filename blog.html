(1)-What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a webpage as a tree of objects that can be manipulated using JavaScript or other programming languages. The DOM allows developers to interact with HTML and XML documents by accessing, modifying, or deleting elements dynamically.
Key Points about the DOM:
Tree Structure:
The DOM represents the document (usually an HTML document) as a tree, where every element, attribute, or piece of content is a node. At the top is the document object, which represents the entire page. Dynamic Interaction:
The DOM allows developers to change the structure, style, and content of a webpage without reloading the page. You can add or remove elements, change their attributes, modify styles, etc. Browser-Generated:
Browsers automatically create the DOM when they load a webpage by parsing the HTML or XML document. Programming Language Independent:
The DOM is language-agnostic, meaning it can be manipulated using different programming languages. However, JavaScript is the most common language used to interact with the DOM in web development. Hierarchy:
The DOM follows a hierarchical structure, where elements are parents, children, and siblings. For example, an HTML body tag is a parent of the div tag, and two div tags inside the body tag are siblings.

How do you select an element from the DOM?

To select an element from the DOM (Document Object Model) in JavaScript, you can use various methods. Here are the most common ones:
1. document.getElementById():
const element = document.getElementById('myElement');
2. document.getElementsByClassName():
Selects all elements with a specific class name.
const elements = document.getElementsByClassName('myClass');
3. document.getElementsByTagName():
Selects all elements with a specific tag name (like div, p, etc.)
const elements = document.getElementsByTagName('div');
4. document.querySelector():
Selects the first element that matches a specified CSS selector.
const element = document.querySelector('.myClass');
5. document.querySelectorAll():
elects all elements that match a specified CSS selector.
onst elements = document.querySelectorAll('.myClass');

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in JavaScript that allows you to handle events at a higher level in the DOM rather than attaching event listeners to individual elements. It leverages the concept of event bubbling, where an event triggered on a child element propagates up to its parent elements. By using event delegation, you can attach a single event listener to a parent element and manage all its child elements' events through it.
How Event Delegation Works:
Event Bubbling: When an event (like a click) occurs on an element, it first triggers on that element and then propagates or "bubbles up" to its ancestors, eventually reaching the root element (document).
Listening at a Higher Level: Instead of attaching event listeners to each child element, you attach a single event listener to a parent or container element. The event listener then captures the events that bubble up from the child elements.
event.target: The event.target property tells you which element triggered the event. By checking event.target inside the event handler, you can determine which child element the event occurred on and take the appropriate action.

How do you manipulate an element's attributes and styles using the DOM?

Manipulating an element's attributes and styles in the DOM can be done using JavaScript. You can use specific methods and properties to modify both HTML attributes and CSS styles dynamically.
1. Manipulating HTML Attributes HTML attributes (like src, href, id, class, etc.) can be accessed and modified using the following methods and properties:
a. getAttribute() and setAttribute() getAttribute(attributeName): Retrieves the value of the specified attribute. setAttribute(attributeName, value): Sets a new value for the specified attribute. Accessing and Modifying Attributes Directly
You can also modify some attributes directly using JavaScript object properties. For example, id, className, href, value, etc. Manipulating CSS Styles You can modify the inline styles of an element using the style property or by dynamically adding/removing CSS classes.
a. Using the style Property The style property allows you to directly set individual CSS properties for an element. The syntax is camelCase, meaning properties like background-color become backgroundColor.
b. Adding and Removing Classes Instead of modifying styles directly, it's often better to toggle CSS classes. This keeps your styling in CSS files, making your code more maintainable. classList.add(): Adds a class to the element. classList.remove(): Removes a class from the element. classList.toggle(): Toggles a class (adds if not present, removes if present).
3. Reading Computed Styles You can read the final, computed styles of an element (including styles set by CSS or inherited from parent elements) using getComputedStyle().
